# Changelog

Все значимые изменения проекта **OTK DXF Inspector** документируются здесь.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/).

---

## [0.5.1] — 2026-03-14

Камера, панель слоёв и контекстное меню.

### Changed

- При смене видимых слоёв сохраняются масштаб и панорама DXF-вида
- Панель «Слои» скрывается, если в чертеже один слой
- Действия «Редактировать» и «Удалить» для размеров перенесены в контекстное меню по правому клику на карточке

---

## [0.5.0] — 2026-03-14

Слои в DXF-просмотре для удобного выбора размеров на перегруженных чертежах.

### Added

- **Слои в DXF-просмотре**: переключение видимости слоёв на странице «Шаблон контроля»; фильтрация отрисовки и выбора размера (pick mode) по включённым слоям; панель с чекбоксами по слоям из распознанных entity

---

## [0.4.0] — 2026-03-13

Завершение MVP: персистентное хранение, история, настройки, графики, тесты.

### Added

- **Персистентное хранение**: SQLite (`data/otk.db`) — проекты, контрольные точки, история изменений; данные сохраняются между запусками
- **История изменений**: `HistoryService`, страница «История» с фильтром по проекту; логирование добавления/изменения/удаления точек
- **Настройки**: страница настроек (папка данных, оператор, тема), сохранение в `config.json` через `ConfigService`
- **Графики аналитики**: matplotlib — диаграмма отклонений по размерам с линиями допуска и pie-диаграмма распределения статусов (Норма / Внимание / Брак)
- **Экспорт**: колонка «Статус» и итоговая строка в Excel; оператор и дата в PDF/печати; экспорт в CSV
- **DXF pick mode**: подсветка ближайшей entity при наведении, overlay-подсказка «Кликните на размер на чертеже»
- **Юнит-тесты**: pytest — ProjectService, DxfParser, ExportService, ConfigService
- **Качество кода**: mypy и ruff в `pyproject.toml`, исправление типов и стиля

### Changed

- Кнопки «Сохранить» и «Печать» на странице замеров подключены к сохранению точек и печати отчёта
- Кнопка «Импорт DXF» открывает диалог создания проекта с предзаполненным путём к файлу

---

## [0.3.1] — 2026-03-13

Исправление отображения чертежей.

### Fixed

- Размерные надписи (DIMENSION) больше не отображаются зеркально — добавлен `scale(1, -1)` к виду DXF-просмотрщика, как того ожидает бэкенд ezdxf (PyQtBackend)
- Согласованы координаты pick mode и маркеров с новой системой координат (scene = DXF)
- Выноски (callout) корректно отображают текст при перевёрнутой оси Y

---

## [0.3.0] — 2026-03-12

Точность маркеров, подвижный кроссхейр, UI-полировка.

### Changed

- Позиция маркеров DIMENSION теперь вычисляется по реальным точкам замера (defpoint2/defpoint3 для линейных, defpoint для радиусных/диаметральных) вместо text_midpoint
- Кроссхейр маркера стал перетаскиваемым — можно вручную скорректировать привязку к чертежу
- Leader-линия обновляется при перемещении любого из концов (и кроссхейра, и лейбла)
- Фон DXF-просмотрщика заменён на тёмный (#1e2030) для лучшей видимости
- Кнопка «+ Размер» переименована в «Добавить размер» для консистентности
- DxfEntityInfo расширен полями def_x1/y1/x2/y2 — точки, между которыми производится замер
- LINE-парсер сохраняет start/end как def-точки

### Fixed

- Подавлен спам fontTools DEBUG в логах при загрузке DXF с шрифтами

---

## [0.2.0] — 2026-03-12

Редактирование размеров, умный импорт, экспорт.

### Added

- Редактирование существующего размера: кнопка «Редактировать» + двойной клик по карточке открывает диалог в режиме edit
- `ProjectService.update_point()` — обновление полей контрольного размера по номеру
- `ExportService` — экспорт в Excel (openpyxl), PDF (QPrinter), печать (QPrintDialog), протокол контроля
- Кнопки экспорта на странице «Аналитика»: протокол, PDF, Excel, печать — с QFileDialog и обработкой ошибок
- Реальные stat cards на странице «Аналитика»: количество замеров, среднее отклонение, количество брака — вычисляются из данных проекта
- `StatCard.set_value()` — динамическое обновление значений
- Зависимость `openpyxl>=3.1` в requirements.txt

### Changed

- Автоимпорт теперь только DIMENSION-сущности (`auto_import_dimensions`), а не все entity — для hard.dxf 13 размеров вместо 125
- Если DIMENSION не найдены — ручной режим без импорта LINE/ARC/CIRCLE
- `AnalyticsPage` принимает `ProjectService`, `refresh()` вызывается при переключении на вкладку

### Removed

- Страница «Отчёты» — кнопки экспорта перенесены в «Аналитику»
- Пункт «Отчеты» из бокового меню

### Fixed

- Краш `DXFAttributeError` при парсинге DIMENSION из hard.dxf — добавлен fallback через MTEXT virtual entities
- Кнопка «Удалить размер» — компактный стиль, ограничена по ширине, выровнена вправо
- Стиль `#dangerButton` в QSS — уменьшены отступы и border-radius

## [0.1.0] — 2026-03-12

Первый рабочий прототип (MVP).

### Added

- Десктопное приложение на PySide6 с навигацией через боковое меню (sidebar)
- **Проекты** — создание, удаление, выбор текущего проекта; таблица со списком проектов и stat-карточки
- **Шаблон контроля** — интерактивный DXF-просмотрщик (ezdxf + PyQtBackend) с масштабированием и панорамированием
- **Pick mode** — привязка контрольного размера к фигуре на чертеже кликом (CIRCLE, ARC, LINE, LWPOLYLINE, DIMENSION)
- Цветные пронумерованные маркеры на чертеже, соответствующие контрольным точкам
- Карточки контрольных размеров с панелью свойств выбранного размера
- Диалог добавления размера с предзаполнением из выбранной entity (тип, значение)
- **Измерения** — таблица ввода замеров с автоматическим расчётом отклонения и статуса (Норма / Внимание / Брак)
- Страницы-заглушки: Аналитика, История, Настройки
- Доменная модель: `Project`, `ControlPoint` (с `deviation`, `status`), `DxfEntityInfo`
- Слой сервисов: `ProjectService` (CRUD проектов и точек), `DxfParser` (парсинг DXF-файлов, рекурсивное раскрытие INSERT)
- In-memory репозиторий с демо-данными для первого запуска
- Логирование по всем слоям (сервисы, инфраструктура, UI) через `logging`
- Архитектура по SOLID/DRY: `BasePage` для страниц, `PointCard` как отдельный компонент, DxfParser вынесен из UI в сервисный слой
- Светлая промышленная тема (QSS)
